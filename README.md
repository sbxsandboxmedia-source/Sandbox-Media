# Sandbox Media Website — Final V1.5 Handover Guide

This is the final integrated static website package for GitHub Pages or Vercel.

## 1. Important rule

Keep the complete `assets` folder. Do not delete or rename the folder.

To replace a placeholder image:

1. Find the exact file path below.
2. Resize and compress your real image.
3. Give it the exact same filename.
4. Upload it to the same folder and replace the old placeholder.
5. No HTML, CSS or JavaScript change is required.

Use lowercase filenames with hyphens only. Recommended format: **WebP**, quality **80–85%**, sRGB color profile.

## 2. Root files

## 5. Portfolio gallery naming

For every non-case-study portfolio item, keep this structure:

```text
ITEM-ID.webp
ITEM-ID-gallery-1.webp
ITEM-ID-gallery-2.webp
ITEM-ID-gallery-3.webp
ITEM-ID-gallery-4.webp
ITEM-ID-gallery-5.webp
ITEM-ID-gallery-6.webp
ITEM-ID-gallery-7.webp
```

`gallery-1` is the large main image. Images 2–7 appear in the two-column supporting gallery.

Project IDs, categories, titles and folder paths are defined in `portfolio-work-data.js`.

Website case studies use image names defined in `portfolio-data.js` and open through:

```text
case-study.html?id=PROJECT-ID
```

## 6. Products

Products are managed in `product-data.js`.

To add a product:

1. Copy an existing complete product object.
2. Change its unique `id`.
3. Update title, category, description, status, price, links, features and FAQs.
4. Upload the images using paths entered in the product object.
5. Open it with `product.html?id=YOUR-PRODUCT-ID`.

## 7. Blog

Blog articles are managed in `blog-data.js`.

Recommended folder:

```text
assets/images/blog/ARTICLE-ID/
```

Recommended files:

```text
ARTICLE-ID-cover.webp          1600 × 1000 px
ARTICLE-ID-content-1.webp      1400 × 900 px
```

Article URL:

```text
blog-post.html?id=ARTICLE-ID
```

## 8. Intro animation

The homepage intro uses:

```text
assets/logos/sandbox-logo.png
intro-animation.js
```

It runs once per browser session. To test it again in the browser console:

```javascript
sessionStorage.removeItem("sandboxIntroShown");
location.reload();
```

## 9. Skip-to-content accessibility link

The link is now completely hidden during normal browsing and appears only when focused with the keyboard `Tab` key.

Do not delete it. It supports keyboard and screen-reader users and improves accessibility.

## 10. Text that should never remain public

Developer instructions such as the following have been removed from public pages:

- replace before launch
- upload images using README

Future developer notes should be written only in this README, not inside visible website sections.

## 11. SEO launch checklist

Before launch:

1. Replace `https://yourdomain.com/` everywhere.
2. Confirm canonical URLs.
3. Update `sitemap.xml` with the final domain.
4. Update `robots.txt` sitemap URL.
5. Add Google Search Console verification.
6. Add analytics only after consent requirements are reviewed.
7. Compress all real images.
8. Replace all `example.com` demo links.
9. Review titles and descriptions for every page.

## 12. GitHub Pages deployment
